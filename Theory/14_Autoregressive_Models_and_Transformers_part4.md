# 47. The Attention Mechanism

The limitations of RNNs and encoder–decoder models led researchers to ask an important question:

> Instead of compressing an entire sequence into one hidden state, can every token directly examine all the other relevant tokens?

The **Attention Mechanism** provides exactly this capability.

Attention allows the representation of one token to be constructed using information from other tokens in the sequence.

---

# 48. Intuition: Looking Back at the Relevant Words

Consider the sentence:

```text
The animal did not cross the road because it was tired.
```

What does the word

```text
it
```

refer to?

Most likely:

```text
the animal
```

To understand `it`, we must examine other words in the sentence.

The model should pay more attention to:

- animal
- tired

and less attention to:

- road
- cross

Attention allows the model to assign different levels of importance to different tokens.

---

# 49. Attention as Weighted Information Gathering

Suppose we are computing a contextual representation for one token.

The model does three things:

1. It compares the token with every token in the sequence.
2. It determines how relevant each token is.
3. It combines information from the tokens according to those relevance scores.

Conceptually:

```text
Token of Interest

↓

Compare with Every Token

↓

Calculate Relevance Scores

↓

Combine Information

↓

Context-Aware Representation
```

The result is no longer an isolated token representation.

It is a representation that includes context.

---

# 50. From Tokens to an Input Matrix

Assume that a sequence contains `T` tokens.

Each token is represented by a vector of length `D`.

The complete sequence can therefore be stored in a matrix:

```text
X has shape T × D
```

For example:

```text
Sequence Length T = 4

Embedding Dimension D = 3
```

Then:

```text
X =

Token 1 → [x₁₁, x₁₂, x₁₃]

Token 2 → [x₂₁, x₂₂, x₂₃]

Token 3 → [x₃₁, x₃₂, x₃₃]

Token 4 → [x₄₁, x₄₂, x₄₃]
```

Each row corresponds to one token.

---

# 51. Query, Key, and Value

The attention mechanism creates three different projections of every token.

They are called:

- Query
- Key
- Value

The names may initially appear confusing.

The important point is that they are simply three learned linear transformations of the input.

The equations are:

`Q = XWQ`

`K = XWK`

`V = XWV`

where:

- `X` is the input token matrix.
- `WQ` is the learnable query-projection matrix.
- `WK` is the learnable key-projection matrix.
- `WV` is the learnable value-projection matrix.

The entries of `WQ`, `WK`, and `WV` are learned during training.

---

# 52. A Search-System Analogy

Imagine searching for a book in a library.

Your search request is:

```text
Beginner-friendly book about probability for machine learning
```

This is the **Query**.

Every book has searchable information such as:

- title
- topic
- author
- keywords

These behave like **Keys**.

The actual content of each book behaves like the **Value**.

Conceptually:

```text
Query

↓

Compare with Book Keys

↓

Find Relevant Books

↓

Retrieve Their Values
```

Attention follows the same idea.

The query asks:

> Which tokens are relevant to the current token?

The keys are used for matching.

The values contain the information that will be combined.

---

# 53. Query Intuition

The query represents:

> What information is this token looking for?

Suppose the current token is:

```text
bank
```

Its query might be looking for context that answers:

- Is this a financial bank?
- Is this a river bank?
- What nearby words clarify the meaning?

The query changes depending on the token and the learned projection.

---

# 54. Key Intuition

A key represents:

> What kind of information does this token contain?

Suppose the surrounding words are:

```text
money

river

withdraw

water
```

Their keys help determine whether they are relevant to the query generated by `bank`.

For a financial context:

```text
bank ↔ withdraw

bank ↔ money
```

may receive high similarity scores.

For a geographical context:

```text
bank ↔ river

bank ↔ water
```

may receive high similarity scores.

---

# 55. Value Intuition

The value contains the information that will actually be used to construct the new representation.

The key helps answer:

> Is this token relevant?

The value answers:

> What information should this token contribute?

After the relevance scores are computed, the value vectors are combined using those scores.

---

# 56. Computing Similarity

To determine whether two tokens are related, attention computes a dot product between a query and a key.

For token `i` and token `j`:

`score(i,j) = qᵢ · kⱼ`

A large positive score suggests that token `j` may be relevant to token `i`.

A smaller or negative score suggests lower relevance.

Important:

> The raw dot product is not a probability.

It may be any real number.

The scores must be normalized before they are used as weights.

---

# 57. Example of Attention Scores

Suppose the sentence is:

```text
The cat drank the milk because it was thirsty.
```

When computing the representation of `it`, the model may produce scores like:

| Token | Raw relevance score |
|---|---:|
| The | 0.1 |
| cat | 4.8 |
| drank | 1.2 |
| milk | 0.9 |
| because | 0.3 |
| it | 2.4 |
| was | 0.8 |
| thirsty | 3.7 |

The scores are not yet probabilities.

They must be transformed into normalized weights.

---

# 58. Why Scale the Dot Product?

Attention divides every dot-product score by the square root of the key dimension.

`scaled_score(i,j) = (qᵢ · kⱼ) / √dₖ`

where `dₖ` is the dimension of the query and key vectors.

Why is this done?

When vector dimensions become large, dot products can also become large.

Very large values can cause the Softmax function to become extremely sharp.

This can lead to:

- unstable gradients,
- nearly one-hot attention weights,
- difficult optimization.

Dividing by `√dₖ` keeps the scale of the scores manageable.

This is why the mechanism is called:

**Scaled Dot-Product Attention.**

---

# 59. Converting Scores into Attention Weights

The scaled scores are passed through the Softmax function.

For token `i`, the attention weight assigned to token `j` is:

`αᵢⱼ = exp(scoreᵢⱼ) / Σₗ exp(scoreᵢₗ)`

After Softmax:

- every weight lies between `0` and `1`,
- all weights in one row sum to `1`.

Example:

| Token | Attention weight |
|---|---:|
| The | 0.01 |
| cat | 0.58 |
| drank | 0.05 |
| milk | 0.03 |
| because | 0.01 |
| it | 0.10 |
| was | 0.03 |
| thirsty | 0.19 |

These weights indicate how much information should be taken from each token.

---

# 60. Constructing the Contextual Representation

Once the attention weights are available, the output representation for token `i` is computed as a weighted combination of all value vectors.

`zᵢ = Σⱼ αᵢⱼvⱼ`

This is the central idea of attention.

The output for one token is not based only on that token.

It is a weighted mixture of information from the sequence.

---

# 61. Full Attention Equation

For all tokens together, the attention operation is:

`Attention(Q,K,V) = softmax(QKᵀ / √dₖ)V`

This equation performs four operations:

1. `QKᵀ` compares every query with every key.
2. Division by `√dₖ` stabilizes the scale.
3. `softmax(...)` converts the scores into normalized weights.
4. Multiplication by `V` combines the value vectors.

Although the equation looks compact, it represents the complete attention process.

---

# 62. Understanding the Matrix Dimensions

Suppose:

- sequence length is `T`,
- query/key dimension is `dₖ`,
- value dimension is `dᵥ`.

Then:

```text
Q has shape T × dₖ

K has shape T × dₖ

V has shape T × dᵥ
```

The similarity matrix is:

```text
QKᵀ has shape T × T
```

Why `T × T`?

Because every token is compared with every token.

After multiplying the attention matrix by `V`:

```text
(T × T) × (T × dᵥ) = T × dᵥ
```

The final output contains one new representation for every input token.

---

# 63. Why Attention Includes Context

Before attention, each token is projected independently.

After attention, every output token contains information collected from other tokens.

Consider:

```text
I deposited money in the bank.
```

The representation of `bank` can receive information from:

- deposited
- money

Now consider:

```text
We sat on the bank of the river.
```

The representation of `bank` can receive information from:

- river
- sat

Therefore, the same word can receive different representations in different sentences.

This is called a **contextual representation**.

---

# 64. What Is Self-Attention?

When the queries, keys, and values all come from the same sequence, the mechanism is called:

**Self-Attention**

Here:

`Q = XWQ`

`K = XWK`

`V = XWV`

All three are derived from `X`.

The word `self` means that the sequence is attending to itself.

Each token asks:

> Which other tokens in this same sequence are useful for understanding me?

---

# 65. Does a Token Always Attend Most Strongly to Itself?

Not necessarily.

The query and key for the same token are produced using different learned matrices.

Therefore:

`qᵢ` and `kᵢ`

are not necessarily identical.

A token may attend strongly to itself, but this is not guaranteed.

For example, the token `it` may attend more strongly to `cat` than to itself if that relationship helps interpret the sentence.

---

# 66. Self-Attention Versus the RNN Hidden State

In an RNN, information travels sequentially:

```text
Token 1 → Token 2 → Token 3 → Token 4
```

For Token 4 to use information from Token 1, that information must pass through all intermediate states.

In self-attention, Token 4 can directly examine Token 1:

```text
Token 4 ─────────→ Token 1
```

This shorter information path makes it easier to learn long-range relationships.

---

# 67. Why Attention Can Be Parallelized

RNNs process tokens one after another.

```text
X₁

↓

X₂

↓

X₃

↓

X₄
```

Self-attention computes the relationships between all tokens using matrix operations.

```text
All Tokens

↓

Matrix Multiplication

↓

All Contextual Representations
```

During training, the tokens can therefore be processed in parallel.

This is one major reason Transformers scale better than recurrent architectures.

---

# 68. The Missing Autoregressive Property

The self-attention mechanism described so far allows every token to examine:

- earlier tokens,
- itself,
- future tokens.

But an autoregressive model must predict the next token using only the past.

For example, when predicting Token 4, the model must not see Token 5.

Otherwise, it would be cheating.

This motivates **Causal Masking**.

---

# 69. Causal Masking

A causal mask prevents each token from attending to future tokens.

Suppose the sequence has four tokens.

The permitted attention pattern is:

```text
Token 1 can attend to: Token 1

Token 2 can attend to: Token 1, Token 2

Token 3 can attend to: Token 1, Token 2, Token 3

Token 4 can attend to: Token 1, Token 2, Token 3, Token 4
```

The model cannot look forward.

---

# 70. Causal Mask Matrix

A causal mask can be represented as:

```text
          Key Position

          1    2    3    4

Query 1   0   -∞   -∞   -∞

Query 2   0    0   -∞   -∞

Query 3   0    0    0   -∞

Query 4   0    0    0    0
```

The mask is added before applying Softmax.

The masked attention becomes:

`MaskedAttention(Q,K,V) = softmax((QKᵀ / √dₖ) + M)V`

where `M` is the causal mask.

---

# 71. Why Use Negative Infinity?

Softmax uses exponentials.

`exp(−∞) = 0`

Therefore, any future-token score receives an attention weight of zero.

Example:

```text
Future Token Score

+

−∞

↓

−∞

↓

Softmax

↓

0
```

This ensures that future information is completely blocked.

---

# 72. Causal Masking During Training and Inference

Causal masking is required during both:

- training,
- inference.

During training, the entire sequence may be processed in parallel, but the mask ensures that every position sees only permitted history.

During inference, the model generates tokens one at a time, so future tokens do not yet exist.

---

# 73. Causal Attention Example

Suppose the training sequence is:

```text
I love machine learning
```

When predicting `love`, the model may use only:

```text
I
```

When predicting `machine`, it may use:

```text
I love
```

When predicting `learning`, it may use:

```text
I love machine
```

Causal masking enforces this rule inside the attention matrix.

---

# 74. Multi-Head Attention

A single attention mechanism produces one type of contextual representation.

Researchers found it useful to compute several attention operations in parallel.

This is called:

**Multi-Head Attention.**

Each attention operation is called a **head**.

---

# 75. Committee-of-Experts Analogy

Imagine several experts reading the same sentence.

One expert focuses on:

- grammar.

Another focuses on:

- meaning.

Another focuses on:

- long-range relationships.

Another focuses on:

- names and entities.

Each expert produces a different interpretation.

The interpretations are then combined.

This is the intuition behind multi-head attention.

---

# 76. How Multi-Head Attention Works

Each head has its own projection matrices.

For head `h`:

`Qₕ = XWQₕ`

`Kₕ = XWKₕ`

`Vₕ = XWVₕ`

The output of one head is:

`headₕ = Attention(Qₕ,Kₕ,Vₕ)`

If there are `H` heads, their outputs are concatenated:

`MultiHead(X) = Concat(head₁, head₂, ..., headH)WO`

where `WO` is another learnable projection matrix.

---

# 77. Why Different Heads May Learn Different Relationships

Since each head has different parameters, each head projects the tokens into a different representation space.

One head may assign high attention to nearby grammatical relationships.

Another may capture distant semantic relationships.

However, this interpretation is not guaranteed.

The exact role of each head emerges from training.

The mathematically important fact is:

> Multiple independent attention projections are learned and then combined.

---

# 78. Is Multi-Head Attention Always Better?

The number of heads is a hyperparameter.

More heads increase the model's representational capacity, but they also increase:

- computation,
- memory usage,
- parameter count.

The best number depends on the task and model size.

There is no universal answer.

---

# 79. Attention Alone Is Mostly Linear

The main attention computation is based on:

- matrix multiplication,
- weighted averaging,
- linear projections.

Except for Softmax, most operations are linear.

To make the model more expressive, Transformer blocks include an additional neural network.

This is called the:

**Feed-Forward Network.**

---

# 80. Position-Wise Feed-Forward Network

After attention, every token representation is passed independently through the same small Multi-Layer Perceptron.

Conceptually:

```text
Token Representation

↓

Linear Layer

↓

Activation Function

↓

Linear Layer

↓

Updated Representation
```

A common form is:

`FFN(z) = W₂ activation(W₁z + b₁) + b₂`

The same Feed-Forward Network is applied to every token position.

---

# 81. Why Add a Feed-Forward Network?

Attention mixes information across tokens.

The Feed-Forward Network transforms each token's features nonlinearly.

The two operations have different purposes:

```text
Attention

↓

Mix Information Across Tokens
```

```text
Feed-Forward Network

↓

Transform Features Within Each Token
```

Together, they create a powerful representation-learning block.

---

# 82. Residual Connections

Transformer blocks also use residual or skip connections.

Instead of replacing the input completely, the model adds the block's input to its output.

Conceptually:

```text
Input X

├───────────────┐
│               │
▼               │
Attention       │
│               │
▼               │
Add ◄───────────┘
```

The operation is:

`Output = X + Attention(X)`

Residual connections help:

- gradients flow through deep networks,
- preserve the original information,
- allow a block to learn only the required change.

---

# 83. Layer Normalization

Transformers commonly use **Layer Normalization**.

Layer Normalization stabilizes the representation of each token by normalizing its features.

Conceptually:

```text
Token Feature Vector

↓

Normalize Features

↓

Stable Representation
```

Normalization improves optimization when many Transformer blocks are stacked.

---

# 84. The Transformer Block

A simplified Transformer block contains:

```text
Input Tokens

↓

Multi-Head Attention

↓

Residual Connection + Layer Normalization

↓

Feed-Forward Network

↓

Residual Connection + Layer Normalization

↓

Output Tokens
```

In a decoder-only language model, the attention is causally masked.

---

# 85. The Complete Block Mathematically

A simplified block may be written as:

`H₁ = LayerNorm(X + MaskedMultiHeadAttention(X))`

`H₂ = LayerNorm(H₁ + FFN(H₁))`

The exact order of normalization differs across architectures.

Some models normalize before each sub-layer, while others normalize after it.

The core components remain the same.

---

# 86. Attention Does Not Know Token Order

Self-attention compares token representations, but by itself it does not know whether a token is:

- first,
- fifth,
- last.

If the token order were rearranged, attention would not automatically know the original positions.

Therefore, positional information must be added explicitly.

---

# 87. Positional Encoding

A **Positional Encoding** is a vector that represents a token's location in the sequence.

Conceptually:

```text
Token Embedding

+

Position Embedding

↓

Position-Aware Token Representation
```

For example:

```text
Word Embedding for "cat"

+

Position Vector for Position 3
```

This tells the model:

- what the token is,
- where the token appears.

---

# 88. Why Not Add the Position as One Number?

Suppose position is simply represented as:

```text
3
```

A neural network may not use one scalar effectively.

Instead, the position is converted into a vector having the same dimension as the token embedding.

This is similar to the time embedding used in diffusion models.

---

# 89. Sinusoidal Positional Encoding

The original Transformer used sine and cosine functions with different frequencies.

For each position, these functions produce a unique vector.

Conceptually:

```text
Position 1 → [sin₁(1), cos₁(1), sin₂(1), cos₂(1), ...]

Position 2 → [sin₁(2), cos₁(2), sin₂(2), cos₂(2), ...]

Position 3 → [sin₁(3), cos₁(3), sin₂(3), cos₂(3), ...]
```

Each position receives a different pattern.

The exact formula is less important initially than the central idea:

> Convert the scalar position into a high-dimensional vector.

---

# 90. Token Embedding Plus Position Encoding

The final Transformer input is:

`InputRepresentation = TokenEmbedding + PositionalEncoding`

Both vectors have the same dimension.

Example:

```text
Meaning of the Word

+

Location of the Word

↓

Complete Input Representation
```

---

# 91. Decoder-Only Transformer Architecture

Most autoregressive language models use a decoder-only Transformer architecture.

The overall workflow is:

```text
Input Token IDs

↓

Token Embeddings

↓

Add Positional Information

↓

Masked Multi-Head Self-Attention

↓

Feed-Forward Network

↓

Repeat Transformer Blocks

↓

Linear Vocabulary Projection

↓

Softmax

↓

Next-Token Probability Distribution
```

The model outputs one probability for every token in the vocabulary.

---

# 92. What Is the Vocabulary?

The vocabulary is the fixed set of tokens the model can produce.

For example:

```text
Vocabulary =

["the", "cat", "dog", "learn", "ing", ".", ...]
```

Modern tokenizers frequently use subword units instead of complete words.

The final output layer has one logit for each vocabulary entry.

---

# 93. Tokenization

Before text enters the Transformer, it must be divided into tokens.

This process is called:

**Tokenization**

A word may remain one token:

```text
apple
```

or be split into multiple tokens:

```text
unbelievable

↓

un

believ

able
```

Subword tokenization allows the model to handle:

- common words,
- rare words,
- new words,
- word variations.

---

# 94. Byte Pair Encoding Intuition

One common tokenization approach is **Byte Pair Encoding**, or BPE.

The basic idea is:

1. Begin with small units such as characters or bytes.
2. Find pairs that frequently occur together.
3. Merge those frequent pairs into new tokens.
4. Repeat.

Example:

```text
l + e → le

le + arn → learn

learn + ing → learning
```

Frequently occurring patterns gradually become tokens.

---

# 95. Training an Autoregressive Transformer

The model is trained to predict the next token.

Suppose the sequence is:

```text
I love machine learning
```

The input and targets are shifted by one position.

```text
Input:

I        love        machine

Target:

love     machine     learning
```

This method is commonly called **Teacher Forcing**.

---

# 96. Teacher Forcing

During training, the model receives the correct previous tokens.

It does not have to use its own earlier predictions.

For example:

```text
Input:  I love machine

Target: love machine learning
```

The model simultaneously learns:

- predict `love` from `I`,
- predict `machine` from `I love`,
- predict `learning` from `I love machine`.

Causal masking prevents each position from seeing its target or future tokens.

---

# 97. Autoregressive Training Objective

For a sequence `X₁, X₂, ..., XT`, the model maximizes:

`Σₜ log Pθ(Xₜ | X₁, X₂, ..., Xₜ₋₁)`

Equivalently, the training loss minimizes the negative log-likelihood:

`Loss = −Σₜ log Pθ(Xₜ | X₁, X₂, ..., Xₜ₋₁)`

This is the same maximum-likelihood principle used throughout the course.

Only the model distribution has changed.

---

# 98. Why Cross-Entropy Loss Appears

The next token is selected from a finite vocabulary.

Therefore, next-token prediction is a multi-class classification problem.

The model outputs:

```text
Probability of Token 1

Probability of Token 2

...

Probability of Token V
```

where `V` is the vocabulary size.

The appropriate loss is **Categorical Cross-Entropy**.

For the correct target token `y`:

`Loss = −log Pθ(y | previous tokens)`

If the model gives the correct token high probability, the loss is small.

If it gives the correct token low probability, the loss is large.

---

# 99. One Training Example Produces Many Targets

A sequence of length `T` does not produce only one training example.

It produces up to `T − 1` next-token prediction targets.

Example:

```text
Sequence:

Deep learning is powerful
```

Training relationships:

```text
Deep → learning

Deep learning → is

Deep learning is → powerful
```

This is why large text corpora provide enormous quantities of self-supervised training data.

---

# 100. Why It Is Called Self-Supervised Learning

Human annotators do not need to manually label the next word.

The sequence itself provides the target.

Input:

```text
Machine learning is
```

Target:

```text
useful
```

The target comes directly from the original text.

Therefore, next-token training is usually called:

**Self-Supervised Learning**

rather than traditional supervised learning.

---

# 101. Inference: Generating Text

During inference, the model starts with a prompt.

Example:

```text
Machine learning is
```

The model produces a probability distribution over the next token.

Suppose:

| Token | Probability |
|---|---:|
| useful | 0.42 |
| difficult | 0.21 |
| changing | 0.10 |
| banana | 0.001 |

One token is selected.

The selected token is appended to the sequence.

The process repeats.

---

# 102. Autoregressive Generation Loop

```text
Prompt

↓

Predict Next-Token Distribution

↓

Select One Token

↓

Append the Token

↓

Run the Model Again

↓

Predict the Next Token

↓

Repeat Until Stopping Condition
```

This sequential generation process explains why autoregressive text generation is naturally slower than fully parallel processing.

---

# 103. Greedy Decoding

The simplest decoding method is:

> Select the token with the highest probability.

This is called **Greedy Decoding**.

Example:

| Token | Probability |
|---|---:|
| useful | 0.42 |
| difficult | 0.21 |
| changing | 0.10 |

Greedy decoding selects:

```text
useful
```

Greedy decoding is deterministic.

The same prompt and the same model produce the same output.

---

# 104. Limitation of Greedy Decoding

The most probable token at the current step may not lead to the best complete sentence.

Choosing locally optimal tokens does not guarantee a globally optimal sequence.

Greedy decoding can also produce:

- repetitive text,
- predictable text,
- low diversity.

---

# 105. Beam Search

Beam Search keeps several promising sequences instead of only one.

Suppose the beam width is `3`.

The model stores the best three candidate sequences at each step.

Conceptually:

```text
Candidate 1

Candidate 2

Candidate 3

↓

Expand Each Candidate

↓

Keep the Best Three New Sequences
```

Beam Search is commonly useful when a stable, high-probability output is preferred.

---

# 106. Random Sampling

Instead of always choosing the highest-probability token, we may sample from the complete probability distribution.

Example:

| Token | Probability |
|---|---:|
| useful | 0.42 |
| difficult | 0.21 |
| changing | 0.10 |

The token `useful` is most likely, but `difficult` may sometimes be sampled.

Sampling creates more varied outputs.

However, unrestricted sampling may occasionally select very unlikely tokens.

---

# 107. Top-K Sampling

Top-K sampling keeps only the `K` most probable tokens.

For example, if `K = 3`:

| Token | Probability |
|---|---:|
| useful | 0.42 |
| difficult | 0.21 |
| changing | 0.10 |

All other tokens are removed.

The remaining probabilities are normalized again, and the model samples from them.

This prevents extremely unlikely tokens from being selected.

---

# 108. Top-P or Nucleus Sampling

Top-P sampling selects the smallest set of tokens whose cumulative probability reaches a threshold `P`.

Suppose `P = 0.90`.

The model keeps adding the most likely tokens until their combined probability is at least `0.90`.

Unlike Top-K, the number of retained tokens can change at every step.

If the distribution is confident, only a few tokens may be needed.

If the distribution is uncertain, more tokens may be included.

---

# 109. Temperature

Temperature controls how sharp or flat the next-token probability distribution becomes.

The modified probability uses:

`Softmax(logits / τ)`

where `τ` is the temperature.

---

## Low Temperature

When `τ < 1`:

- the probability distribution becomes sharper,
- high-probability tokens become more dominant,
- output becomes more deterministic.

---

## High Temperature

When `τ > 1`:

- the probability distribution becomes flatter,
- lower-probability tokens receive more chance,
- output becomes more diverse and unpredictable.

---

## Temperature Intuition

Imagine choosing food from a restaurant.

Low temperature:

```text
Always order your favorite dish.
```

High temperature:

```text
Try something unusual from the menu.
```

---

# 110. Why Attention Has Quadratic Cost

The attention matrix has shape:

```text
T × T
```

because every token is compared with every token.

Therefore, the computation and memory grow approximately quadratically with sequence length.

`Attention Cost ∝ T²`

If the sequence length doubles:

```text
T → 2T
```

the size of the attention matrix becomes approximately four times larger.

This is one of the major limitations of standard attention.

---

# 111. Context Window

The **Context Window** is the maximum number of tokens a model can process in one sequence.

A model may support a maximum context of:

```text
T tokens
```

Shorter sequences may be padded or batched using attention masks.

Longer sequences must be:

- truncated,
- divided into chunks,
- or handled using architectures designed for longer context.

The context length is an important model configuration and training choice.

# 112. Vision Transformers

Transformers are not limited to text.

An image can also be converted into a sequence.

This leads to the **Vision Transformer**, or ViT.

# 113. Converting an Image into Tokens

Suppose an image is divided into small patches.

```text
Image

↓

Patch 1

Patch 2

Patch 3

...

Patch T
```

Each patch is flattened and projected into a vector.

Each image patch becomes one token.

The sequence of patch tokens is then passed through a Transformer.

---

# 114. Vision Transformer Pipeline

```text
Input Image

↓

Divide into Fixed-Size Patches

↓

Flatten Each Patch

↓

Project Each Patch into an Embedding

↓

Add Positional Information

↓

Transformer Blocks

↓

Output Representation
```

The Transformer processes image patches in the same general way that it processes word tokens.

# 115. Transformer Versus Generative Model

An important distinction from Prathosh's lecture is:

> A Transformer is an architecture, not the probability model itself.

The underlying generative model for most language models is:

```text
Autoregressive Model
```

The Transformer is the neural-network architecture used to represent:

`Pθ(Xₜ | X₁, ..., Xₜ₋₁)`

This is similar to how a U-Net is an architecture used inside a diffusion model.

# 117. Common Misconceptions

## Misconception 1

**Attention and Transformer mean the same thing.**

They do not.

Attention is one operation inside a Transformer.

A Transformer also includes:

- Feed-Forward Networks,
- normalization,
- residual connections,
- positional information.

---

## Misconception 2

**A Transformer is itself a generative modeling objective.**

It is not.

The Transformer is an architecture.

The autoregressive probability factorization defines the generative model.

---

## Misconception 3

**One token always equals one word.**

It does not.

A token may be:

- a word,
- a subword,
- punctuation,
- a byte,
- part of a number.

---

## Misconception 4

**The query, key, and value are manually designed features.**

They are not.

They are learned projections created using `WQ`, `WK`, and `WV`.

---

## Misconception 5

**Multi-head attention guarantees that every head learns a human-interpretable concept.**

It does not.

Different heads learn different projections, but their exact meanings may not always be interpretable.

---

## Misconception 6

**Self-attention automatically knows token order.**

It does not.

Positional information must be added separately.

# Chapter Summary

In this chapter, we studied the foundations of autoregressive language models and Transformers.

We learned that:

- Sequential data requires order-aware models.
- Autoregressive models generate one token at a time.
- The Chain Rule factorizes a sequence probability into next-token conditional probabilities.
- LPC and N-gram models were early autoregressive approaches.
- Neural language models replaced counting with learned nonlinear functions.
- RNNs introduced hidden states and parameter sharing.
- LSTMs improved memory but remained sequential.
- Encoder–Decoder models compressed the input into a bottleneck representation.
- Attention allowed each token to directly use information from other tokens.
- Query, Key, and Value are learned linear projections.
- Scaled dot-product attention constructs contextual token representations.
- Causal masking prevents access to future tokens.
- Multi-head attention learns several attention projections in parallel.
- Transformer blocks combine attention, Feed-Forward Networks, residual connections, and normalization.
- Positional encodings provide information about token order.
- Transformers are trained using next-token prediction and cross-entropy loss.
- Text generation may use greedy decoding, Beam Search, Top-K, Top-P, and temperature.
- Standard attention has quadratic complexity in the sequence length.
- Vision Transformers treat image patches as tokens.

# Key Takeaways

✅ A Transformer is an architecture used to model autoregressive conditional distributions.

✅ Attention creates contextual representations by combining value vectors using learned relevance weights.

✅ `Q`, `K`, and `V` are learned projections of the input.

✅ The core attention equation is:

`Attention(Q,K,V) = softmax(QKᵀ / √dₖ)V`

✅ Causal masking ensures that the model cannot see future tokens.

✅ Positional information is necessary because attention alone does not preserve sequence order.

✅ Transformer training remains Maximum Likelihood Estimation implemented through next-token cross-entropy loss.

✅ Modern LLM generation remains autoregressive: one new token is selected and appended at each step.
